Performing Advanced Searches

The search screens allow an operator to search the DHS database for baggage, flight, and alarm records that meeting a given set of criteria. The data is returned as an ad-hoc report. Each of the search screens behaves in the same manner: search criteria are selected, and the results are displayed once the search is executed.

Once a search has been executed, the search results can be sorted, printed, or exported to file (CSV, PDF, or Excel). The search criteria can also be modified at any time.

The printout includes a title (i.e. the name of the search screen), the date and time when the search was printed, and all records including column headers.

It is possible to display and print all columns from the associated table when the results are returned. However, this may be more information than is needed by the operator. Additionally, when many columns are printed, the results may be difficult to use, as the details for a single record can be split across multiple pages. Therefore, the operator will be able to choose which columns are displayed. The columns that are exported or printed are the same columns that are displayed on the screen.

The following searches are available:

  • Activity Log Search – Allows the user to search through the log of activities from the SAC. This includes events such as errors in processing messages from other systems (e.g. BSM, FIDS), bags being sorted, bags being released at MEC, etc. The information available is a less detailed version of log files generated by the various services/applications. See section 10.3.7 Activity Log Search.
  • Alarm History Search – Allows the user to search through the HMI Alarm History Database (e.g. jam events, E-Stop events, etc.). See section 10.3.3 Alarm History Search.
  • Baggage Tag Search – Allows the user to search for an IATA baggage tag number. Detailed information is provided for the associated passenger, flight, sort decisions, bag tag number, etc. See section 10.3.4 Bag Tag Search.
  • Baggage Transaction Search – Allows the user to search for baggage transactions (e.g. scanned at ATR, cleared at ISD, sorted to destination, etc.) associated with a specific baggage tag number. See section 10.3.5 Bag Transaction Search.
  • Audit Log Search – Allows the user to search user actions in the system. This includes actions such as users logging in and out of applications, users modifying system configuration (e.g. virtual to physical pier changes, flight schedule changes, etc.). See section 10.3.8 Audit Log Search.
  • Raw Message Search – Allows the user to search through the strings of raw data that are received from the airport host computers. See section 10.3.6 Raw Message Search.

Advanced Searches allow users to search for baggage records using AND/OR statements. Users will be able to combine multiple search criteria using these logical operators to narrow their searches. These search functions can be accessed by clicking the Main Menu icon (), then clicking Search in the SmartSuite Enterprise Web Client.

Generating an Advance Search

To generate a logical search, enter the desired criteria into the designated fields and click Search.

Most advanced searches only require a time/date range, but additional search terms can be added to narrow the results. You can add additional search terms by clicking in the blank (Select Criteria) field in the Search Criteria section (see callout ).

Advanced Search - Adding Search Criteria

When you click a blank (Select Criteria) field, a list of available search terms displays (see callout ).

Selecting a Search Criterion

Selecting a search term from the list will display any fields associated with that search term, such as a text field or a date selector (see callout ). You can remove search terms by clicking the associated Delete icon () in the top right corner of the search term.

New Search Term Added

Clicking Search will run the query and the search results will display below the Search Criteria section. The Search Criteria section will also minimize. Clicking the Search Criteria link in the blue bar will expand the section.

Figure 213: Search Results Screen

SmartSuite will remember which search terms you selected and display them for you by default the next time you view the same Advanced Search page. To reset the search terms to the defaults, click the More Actions menu () in the Search Criteria section and then click Reset.

You can click column headers to sort the search results. Arrows beside the column indicate whether results are sorted in ascending or descending order.

Adding Additional Queries

Click the OR button located in the bottom left corner of the Search Criteria window section to add a second query to the search. When two queries are present, the search will return results that match either the first set of criteria or the second set of criteria.

Additional Queries

Cloning, deleting, and resetting search criteria can be done via the More Actions menu () in the Search Criteria section. Note that the column width of the generated search can be expanded/condensed for improved readability.

Configuring Search Result Columns

Clicking the Gear icon () at the top of the screen allows the user to configure which columns display in the search results.

To remove specific columns from the results, click the desired item and drag it to the Hidden Columns section. To keep a column visible on the screen even when you scroll to the right, drag it to the Fixed Columns section. Click OK to keep your changes. To return the table to its original configuration, click Restore Defaults then click OK.

Clicking the Download icon () at the top of the screen allows the user to download the report (PDF, Excel, and CSV format). Clicking the Email icon () allows the user to send a copy of the report (in one of the previously mentioned formats) to a recipient.

Notes:

  • A minimum and maximum date/time offset will be configured by Brock Solutions for Advanced Searches prior to installation. This parameter is used to prevent searches from being queried for periods too far in the past or too far in the future to be useful.
