Virtual to Physical Mapping

Pier assignments are managed through a Virtual to Physical Pier mapping table. Instead of assigning a flight to a physical pier (which could become unavailable), flights are assigned to a virtual pier. Usually the virtual pier destinations are identical to the physical pier destinations (e.g. the virtual pier "MU01" would be assigned to the physical pier "MU01"). However, in cases of equipment failure, flight cancellations, etc. all bags can be easily rerouted by "remapping" the "MU01" virtual pier to another physical pier (e.g. MU02).

Virtual Pier Assignments are also used by the system to handle special cases such as No Read bags. These special virtual piers cannot be assigned to a flight, but they can be assigned to route bags to a specific destination. For example, in a system that has a Manual Encoding Station, any "No Read" or "Reject" bags are normally sent to the MEC, but these locations could also be re-mapped elsewhere.

Special Virtual Piers are configured by the system administrator and may include the following:

  • GlobalEarly – This is the virtual sort destination for any bags that are checked in during the "early" time-period (i.e. before the flight's open time) if an Early Pier is not assigned to the flight. For more information. For more information, see section 4.3.1 Flight Times.
  • GlobalLate – This in the virtual sort destination for any bags that are checked in during the "late" time-period (i.e. after a flight's late time but before a flight's close time) if a Late Pier is not assigned to the flight. For more information, see section 4.3.1 Flight Times.
  • GlobalMissed – This is the virtual sort destination for any bags that are checked in during the "missed" time-period (i.e. after a flight's close time) if a Missed Pier is not assigned to the flight. For more information. For more information, see section 4.3.1 Flight Times.
  • CancelledFlight – This is the virtual sort destination for any bags assigned to a flight that has been cancelled.
  • Problem Pier – This virtual sort destination is used for the Problem Pier Quick release button on the MEC.
  • NoRead – This is the virtual sort destination for any bags that could not be read by the Automatic Tag Reader (ATR).
  • Reject – This is the virtual sort destination for any bags that could not be automatically be sorted by the Automatic Tag Reader (ATR). Examples of such Logical Reject reasons are: No BSM Data, Flight Not in Schedule, Flight Pier Not Assigned, etc.

To access the Virtual to Physical Mapping screen, click the Main Menu icon (), select Sortation, then click Virtual to Physical Mapping.

Note that if a single virtual pier is mapped to multiple physical piers, bags will alternate between the physical piers in round-robin style.

To Configure Virtual Pier Assignments

  1. Select the relevant Physical Pier from the Physical Piers list. The details for that Physical Pier will be displayed.
  1. To add a virtual pier mapping, select a pier from the Add Virtual Pier drop-down list. Note that the pier will automatically be added to the virtual to physical mapping table, but the Virtual to physical mapping must be saved before the change takes effect in the sortation system.
  1. To remove a virtual pier mapping, select the checkbox beside the desired pier(s) and click Remove. Clicking the checkbox in the header row of the Virtual Piers table will select all virtual piers. Note that the pier will automatically be removed from the virtual to physical mapping table, but the virtual to physical mapping must be saved before the change takes effect in the sortation system.
  1. To copy a virtual pier to another physical pier or to move a virtual pier to another physical pier, select the checkbox beside the desired pier(s) and then select the physical pier from the Copy/Move To drop-down list. From the Copy/Move dialog that is displayed, press the Move button to move the mapping or check the Copy box if a round-robin operation is desired.
  1. Once the virtual to physical mapping is configured as desired, click the Save button to save the changes. The changes will take effect within 30 seconds.
