CBIS Daily Summary Report

The CBIS Daily Summary Report provides an overview of the distribution of bags across the various input points. The report also provides information pertaining to the sizing (i.e. In Gauge, Out of Gauge statistics) of the bags entering the CBIS system.

The following selection criteria are available:

  • Start Date and Time
  • End Date and Time

Figure 9‑27: Sample CBIS Daily Summary Report

The following information is included in this report:

CBIS Throughput Section

Row

Description

Total CBIS Baggage Throughput

The total number of bags entering the CBIS. This is the same count of total bags that is seen on the Input Conveyors section of the CBIS Bag Volume Report (see section 9.24 CBIS Bag Volume Report).

Average Time in CBIS

The average time spent by each bag in the CBIS. This is based messages received from the PLC between the time a bag was first tracked in CBIS (e.g. ATR, dimensioner) to the point it either exits CBIS on a clear line, or is removed from the alarm line in CBRA at the BRP. Note that a bag must be tracked 100% of the way through CBIS in order to be included in this calculation.

Bag Volume Section

Column

Description

Number of Bags

In Gauge

The number of bags through the dimensioner that were deemed in gauge. This count is incremented and maintained in the PLC based on information received from the dimensioning system.

Out of Gauge

The number of bags through the dimensioner that were deemed out of gauge. This count is incremented and maintained in the PLC based on information received from the dimensioning system.

No Decision

The number of bags through the dimensioner for which no status of the bag sizing was received from the dimensioning system. This count is incremented and maintained in the PLC.

Total

The total number of bags that passed through the dimensioner.

Percentage of Total Bag Volume

In Gauge

The percentage of all bags through the dimensioner that were in gauge.

Out of Gauge

The percentage of all bags through the dimensioner that were out of gauge.

No Decision

The percentage of all bags through the dimensioner that received no decision.

Total

100% (All bags).

CBIS Faults/Events Section

The following shows the occurrences and duration of faults, events, failsafes and jams within the CBIS system.

Column

Description

Fault/Events

The type of fault or event. The following are available:

Fault – A fault is defined as a cause and is created automatically. Examples include: motor overload, encoder failure, diverter failures, PLC faults, etc.

Event – An event is defined as an effect of a fault and is more created by human intervention. Examples include: e-stop, divert all, motor disconnects, etc.

Failsafe – A failsafe is defined as a condition that is detected at a security sorting device where the possibility exists that a bag was mis-sorted to the clear line. When this occurs, the conveyors around the security sorting device stop so that the bag(s) can be put back into the CBIS system.

Jam – Jams include both hard jams as well as missing bag jams.

Lost in Tracking – Total number of Missing Bag Jam faults in the system.

Machine Fault – Total number of events that cause any ISD machine to not be available to process bags.

Number

The number of that type of CBIS fault/event.

Total Down Time

Cumulative time that each fault/event was active (hh:mm:ss) in the CBIS. Note that if multiple faults/events are active at any given time, the total time includes the sum of the lengths of time each event was active.

Average Time to Clear

Average time to clear the faults/events in the CBIS (hh:mm:ss).

Lost in Tracking

Column

Description

Pre-EDS

The number of Lost In Tracking bags between the start of tracking (typically the ATR or Dimensioner) and the entrance to the EDS machine.

Post-EDS

The number of Lost In Tracking bags between the exit of the EDS machine and either a clear line or into CBRA.

Bag Time in CBIS

The following are based on messages received from the PLC between the time a bag was first tracked in CBIS (e.g. ATR, dimensioner) to the point it either exits CBIS on a clear line, or is removed from the alarm line in CBRA at the BRP. Note that a bag must be tracked 100% of the way through CBIS in order to be included in this calculation.

Column

Description

Minimum

The minimum amount of time any bag spent in the CBIS system.

Maximum

The maximum amount of time any bag spent in the CBIS system.

Average

The average time bags spent in the CBIS system.

Bag Jam Rate

The Bag Jam Rate is calculated by dividing the total number of jams after the CBIS ATRs up to the EDS machines, by the total number of bags that have entered the CBIS area. This does not include any bags that have gone down the Out of Gauge line(s).

Peak Hours in CBIS

Column

Description

Peak Day

This report can be run over multiple days, This will show the Peak Day that the Peak Hour is on.

Peak Hour

The Peak Hour of Bag Activity in CBIS.

Total Bags

The total number of bags in CBIS during the Peak Hour.

Invalid Arrivals

Column

Description

OOG

The invalid arrival rate on the Out of Gauge Line. An invalid arrival is any bag that does not have an Out of Gauge status.

CBRA

The Iinvalid arrival rate in CBRA. This includes all bags that are not any of Alarm, OOG, EDS Error or Timeout. While Alarm and OOG bags are the only valid arrivals, EDS Error and Timeout are not included in the CBRA Invalid Arrival rate per PGDS v6.

Upstream Tracking Accuracy

Column

Description

IATA

The total number of IATA bags sucessfully tracked form the ATR to the EDS machine or ATR to the OOG line.

OOG

The total number of Out of Gauge bags sucessfully tracked form the dimensioner to the OOG line.
