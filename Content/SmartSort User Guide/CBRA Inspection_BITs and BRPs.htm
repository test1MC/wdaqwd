CBRA Inspection (BITs and BRPs)

Baggage Removal Point (BRP) terminals and Baggage Inspection Table (BIT) terminals are in the Checked Baggage Resolution Area (CBRA) and are used to assist users in the inspection of bags that are not cleared automatically by security screening. Bags will also be sent to the CBRA area if they are too large to be processed using the Inline Screening Devices (ISDs) or if they are lost in tracking between being processed by the ISDs and being diverted to the "cleared" portion of the sortation system.

In general terms, the CBRA Inspection process is described below:

  1. A bag is sent to the CBRA area.
  2. The bag is delivered to an available BRP terminal (for details about how the system determines which available BRP is used, refer to Brock Solutions Controls Manual Chapter 3: System Level Controls.
  • In contingency operations, bags will stop at the first BRP station and a bag tag can be printed for each bag.
  1. The agent uses the BRP application to select an available BIT workstation. This is the workstation to which the bag will be transferred.
  • In contingency operations the user can also remove the bag without selecting a BIT workstation.
  1. The agent removes the bag from the conveyor and transfers it to the selected BIT workstation.
  • If the bag is not removed within the configured time limit, the transfer will be cancelled.
  • The transfer can also be cancelled by the agent at any time if the wrong BIT was selected or the bag does not need to be removed.
  1. Once removed from the conveyor, the bag's data displays on the selected BIT terminal.
  1. The agent uses the BIT application to view the associated SVS data for the bag (the information associated with the ISD image).
  1. The agent follows SOP for reconciling the bag by sending it back to be scanned by the ISD machine or dealing with it on the table.
