Bag Tag Scanned

The operator can encode a bag by scanning the barcode on the bag tag with the handheld scanner. If the bag tag can be read by the scanner, a BSM exists for that tag, and the flight data exists for the bag, the bag will automatically be routed to the time-specific pier associated with the flight. The same logic used for sorting bags at the ATR is used at the MEC. When the sort logic runs successfully, the bag will leave the Manual Encoding station automatically (the user will not have to select the bag's destination or press a button to release the bag from the MEC station). The bag transaction data will also be automatically updated on the SAC. The MEC will send the bag's destination to the PLC which will route the bag to the correct sort destination.

This is the preferred method for encoding a bag because it is the easiest method for the user to use, it ensures accurate sort destinations, and it allows for traceability of a bag.

If the barcode cannot be scanned, it can be dispatched using one of the other encoding methods described in the following sections.

If the bag tag barcode can be scanned, but a BSM does not exist for the bag, and no carrier default pier is defined for that bag, the bag is processed according to the Pseudo BSM settings (see section 5.2 Pseudo BSMs). If a carrier default pier is defined for a bag that does not have a BSM, the bag will be sent to the assigned default pier (see section 5.2.1.1 Carrier Make-Up Unit Auto-Release).
