Pseudo BSMs

If a bag tag is scanned or manually keyed and a BSM does not exist for that bag tag, the system can create a pseudo BSM (a BSM created by SmartSort rather than one created by the Airport/Airline). The purpose of creating this Pseudo BSM is to provide traceability for a bag to a sort destination. In addition, if a bag is subsequently seen at an ATR, it can be automatically sorted based on the Pseudo BSM instead of returning to MEC again.

Figure 55: Pseudo BSMs

There are two Pseudo BSM modes: normal mode and strict mode, both of which can be configured using the Web Client (for information about configuring Pseudo BSM modes, see the SmartSort System Administration Manual).

In normal mode, the user will be prompted to encode the bag by flight but will be able to select another encoding method (Pier, Carrier Default Pier, or Quick Release).

In Strict mode, the operator can only encode the bag by Flight (see section 5.2.3 Flight Mode) or by selecting a Quick Release destination (see section 5.2.5 Hot Make-Up Unit Selection).

If Pseudo BSMs are enabled, the carrier default user prompt will be disabled. In this case, bags will automatically be released to the carrier's default pier if a default pier has been configured. For details, refer to section 5.2.1.1 Carrier Make-Up Unit Auto-Release.

If Pseudo BSMs are enabled but the carrier default pier has not been defined, the user must encode the bag using one of the available encoding methods.
