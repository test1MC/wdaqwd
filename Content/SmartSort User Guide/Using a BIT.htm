Using a BIT

Before a bag arrives at the BIT, the BIT Enabled screen displays.

Figure 72: BIT - Enabled

When a bag is transferred to the BIT from a BRP, the information associated with that bag will be displayed in the upper section designated Last Bag from BRP as shown in Figure 73. This information includes:

  • Bag Pseudo ID - The PLC-generated value passed to the ISD to associate with the image (used when no other ID is available).
  • IATA Bag Tag Number - If the bag had a readable barcode, it will be passed to the ISD to associate with the image (if no ATRs are installed upstream of screening, this field will be hidden).
  • RFID Tag Number - If the bag had a readable barcode, it will be passed to the ISD to associate with the image (if no readers are installed upstream of screening, this field will be hidden).
  • ISD Machine - The device that screened the bag.
  • BHS ID for the bag - The PLC ID used for tracking the bag in the system (can be used for bag research from the SmartSuite Enterprise Web Client).
  • Photo Eye - Certain bag statuses displays the photo eye where the bag began to be tracked by the PLC.

Figure 73: BIT - Bag Present

Clicking BAIT transfers the bag's information to the SVS. The bag's data continues to be displayed in the Last Bag from BRP section until the information is transferred. There is no time limit on how long a bag can wait for the BAIT button to be clicked, but the BAIT button must be clicked for the station to become available again.

Once the information has been transferred to the SVS, the data will be cleared from the Last Bag from BRP section and moved to the Last Bag to SVS section in the lower half of the screen.

Figure 74: BIT - Data Sent to SVS

The bag's data will continue to be displayed in the Last Bag to SVS section until a new bag is processed, until the BIT application is disabled and re-enabled, or until the BIT terminal is powered off. This allows TSA personnel to view the data for as long as necessary to process the bag. If the BAIT button was clicked before the SVS was ready to accept data, the Resend button can be used to send the same Bag ID data to the SVS as before. This can be repeated as often as necessary.

Once the bag's data has been transferred using the BAIT function and the data is cleared from the Last Bag from BRP section, the BIT workstation is considered available. A user can transfer another bag from a BRP station to the BIT station so that one bag is in queue while the previous bag is being processed.

BITs can be enabled or disabled by clicking the button in the top right corner of the screen. When the BIT is enabled, the button will read Disable. When disabled, a BIT station is not available to process bags, and the following screen displays:

Figure 75: BIT - Workstation Disabled

The BIT station will be grayed out on the associated BRP screen. Tapping Enable will re-enable the BIT station.
