BSD Tag Printer

In Contingency Mode, the operational staff has determined that the system is not operating correctly but is still somewhat usable. This might be the case when the ISDs are not correctly identifying bags and too many are proceeding to CBRA. In Contingency Mode, the print enable status should be turned on from the HMI (using an appropriately-privileged account).

A single tag printer is provided and is installed on the first BSD on each conveyor line entering CBRA. The printer is intended to be used for contingency purposes only. The CBRA tag printer is enabled or disabled via the HMI workstation in the master control room (see section 8.1.2.16 BSD Pop-Up Window).

Figure 64: BSD - Print Mode Enabled

When tag printing is enabled, all bags must be initially processed from the first BSD. Bags will be allowed to proceed to downstream BSDs or to be removed manually from the conveyor when the Advance button on the BSD is pressed.

An identifying tag can be printed from the BSD workstation when in this mode. This tag will contain the bag's pseudo ID, security status, and the time the tag was printed. Tapping the Print button prints the tag (and can be used multiple times if the printer is not fully supplied with tags).

The data and barcode on the tag should allow users to properly screen the bag (or at least identify it) when the contingency mode problems have cleared.
