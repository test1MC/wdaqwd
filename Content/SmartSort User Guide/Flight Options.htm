Flight Options

The Flight Options section of the screen provides additional rules to determine how flights are rerouted.

  • Allow Reroute to On Hold Flight – If this checkbox is selected, any flight currently on hold will be considered a candidate for reroute. The risk with enabling this reroute rule is that there is no guarantee when the flight will be taken off hold.
  • Only Reroute to the Same Airline – Although it would be normal for a system to define different reroute groups for each airline that uses Dynamic Reroute, there is no constraint in the flight schedule to enforce this rule. To prevent bags from being rerouted across airlines (which might not be supported operationally or financially), this checkbox is normally selected. 
  • Maximum Bags to Reroute – The number of bags allowed to be rerouted to a flight, from any source flight. Once the number of bags previously rerouted to a flight has exceeded the maximum, the flight will no longer be selected as a candidate for dynamic reroute. A value of zero in this field means no maximum is configured.
  • Default Pier – The default sort destination for the reroute group is used to configure the reroute method that will be used by default assuming none of the scenarios below are applicable.
