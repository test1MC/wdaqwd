Flight Properties

There are several flight properties that are available. Each one is described below. Unless otherwise noted, these fields are used for flights in the Active Schedule and in the Template Flight Schedule screen.

Flight Information

  • Airline – The airline/airline name. A list of matching airline names will be displayed as the airline name is typed (the list includes all airlines that have currently been configured. See section 4.2 Configuring Carrier Make-Up Units for details). This field is required.
  • Flight – The flight number. This should be a four-digit numeric value. This field is required. If fewer than four digits are entered, the system will add zeroes to the beginning of the number to create a four-digit flight number.
  • Route – The three-character destination airport code or codes for the flight's destination(s). If the flight travels to more than one airport, the codes should be separated by a comma or a comma and a space (e.g. DEN, ORD, YYZ). This field is required.
  • Gate – The gate number for the flight. The gate number can be alphanumeric. This field is optional.
  • Aircraft –The aircraft type that will be used for the flight. A matching list of Aircraft types will be displayed as the aircraft type is entered. If the aircraft type is not available, contact the system administrator to add the aircraft type to the system. This field is optional.
  • Market – The flight market refers to whether the (next) destination airport is Domestic, Transborder, International, etc. A matching list of markets will be displayed as the market is entered. If the market is not available, contact the system administrator to add the market to the system. This field is optional.
  • Reroute – This is a drop-down menu that contains all of the currently configured Reroute groups. The rules associated with the selected Reroute group will be applied if a bag misses its flight or if the flight is cancelled. For details see section 4.4 Reroute Control. This field is optional.

Flight Times - Active Flight Schedule

  • Scheduled Departure – The scheduled departure date and time for the flight. The scheduled departure field is required. If the scheduled departure is updated, the estimated departure is automatically updated with the same value.
  • Estimated Departure – The estimated/actual departure date and time for the flight. The estimated departure field is required. If the estimated departure is updated, the open, late and close times are automatically adjusted by the same amount of time.
  • Open Time – This field is required and can be either automatically calculated by applying the Airline, Aircraft or Market offset (if available) or by manually entering the open time. The open time must be less than the late time and the estimated departure. Note that this field must contain both the date and the time.
  • Late Time – This field is required and can be either automatically calculated by applying the Airline, Aircraft or Market offset (if available) or by manually entering the late time. The late time must be less than or equal to the close time and greater than the open time. Note that this field must contain both the date and the time.
  • Close Time – This field is required and can be either automatically calculated by applying the Airline, Aircraft or Market offset (if available) or by manually entering the close time. The close time must be greater than or equal to the late time. Note that this field must contain both the date and the time.

Flight Times - Flight Schedule Templates

  • Schedule Departure – The schedule departure date and time for the flight. This field is required.
  • Offset Source – The Open Time, Late Time, and Close Time are specified as offsets from the estimated departure date/time. For example, an offset may be configured to start the open time two hours before a departure, start the late time 45 minutes before departure, and start the close time 15 minutes before the departure. These offsets are configured by a system administrator. The available options are Airline, Aircraft Type, Market and Override. If Airline/Aircraft Type/Market is specified, the times are automatically calculated based on the Airline/Aircraft Type/Market open/late/close offsets. Alternately, the values for the Open, Late and Close times can be manually entered.
  • Open Offset – The number of minutes prior to the departure of the flight that the Open Time will be set to begin to in the Active Schedule.
  • Late Offset – The number of minutes (plus or minus) relative to the departure of the flight that the Late Time will be set to begin in the Active Schedule.
  • Close Offset – The number of minutes (plus or minus) relative to the departure of the flight that the Close Time will be set to begin in the Active Flight Schedule.
  • M/T/W/T/F/S/S - Indicates which days of the week (Monday to Sunday) the flight is scheduled to operate on.
  • Effective Date – The date when the flight will start being added to the active flight schedule (if all other requirements are met, e.g. flight operates on Monday). This field is optional. If left blank, the flight will be added to the active schedule based on the Effective Date of the template where the flight is configured as well as the discontinue date listed for the flight schedule template and for the individual flight within the template (whichever date is earlier).
  • Discontinue Date – The last date when the flight will be added to the active flight schedule. If no date is entered, the flight will continue to be added to the active flight schedule every day that it is scheduled to operate on (using the days of the week checkboxes). This field is optional.

Flight Status - Active Flight Schedule

  • Cancelled – Selecting this checkbox indicates that the flight has been cancelled. This field is optional.
  • On Hold – Selecting this checkbox indicates that the flight has been placed on hold and its bags will sort to its On Time sort destination regardless of time. This field is optional.

Pier Assignments

Each flight is assigned to one or more piers so that bags can be sorted to their proper destination. The flight's On Time Pier is required or a warning and alarm will be generated indicating there are flights that do not have a pier assignment. If a flight's time-based pier is not defined, a bag for that flight will sort to the On Time Pier. For example, an early bag will go to the Early Pier if it is defined; otherwise the bag will sort to the On Time Pier.

Other time-based pier types can be added and used to sort bags based on the flight's scheduled departure time. Some systems have other types that are not time-based (e.g. reroute or transfer). These pier assignments sort bags based on other characteristics.

Pier assignments can also be assigned criteria. If a station and/or class criteria is added to a pier assignment, only bags matching those criteria will sort to the pier. If there are multiple piers of the same type with the same criteria, only the first one created will be used. There is no round-robin on flight piers.

  • Pier Types –This is the time-based pier type associated with the pier number. For a description of flight times and the associated time-based pier assignments, see section 4.3.1 Flight Times. An On Time Pier assignment should be specified for every flight. Other time-based pier assignments (early, late, or missed) are optional.
  • Pier – This is the destination sort pier/carousel for the associated time-based sort pier type.
  • Station – If multiple stations have been set up for a flight's route, they can be selected for this field to provide additional sort possibilities. Piers can be set up to sort bags going to different downline stations using this field (e.g. to separate terminating from transfer bags at the end of the next flight leg).
  • Class – Piers can be set up to sort bags in different classes (e.g. Business Class, Premium, etc.) to different sort destinations using this field. The value entered must match the airlines class designations as sent in a BSM (e.g. Y = Economy for Airline XYZ, so the class would be specified as Y in this field).
