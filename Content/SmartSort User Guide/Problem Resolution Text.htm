Problem Resolution Text

From the Alarm Summary screen or from the Alarm Banner, a user can access Problem Resolution text for any displayed alarm by double-clicking on the alarm. The Problem Resolution text displays in a new browser window.

The Problem Resolution text window appears as follows:

Figure 7‑23: Problem Resolution Text

Note that the Problem Resolution text can also be accessed by clicking the Pr. Resolution button on-screen. Pressing this button opens the Problem Resolution text at the beginning of the document (i.e., without automatically jumping to a specific section of the text). From here, the operator will be able to click on links, matching the alarm types, and jump to the appropriate part of the document.

For each type of alarm, the problem resolution text will normally include the following:

  • Problem Description – One or two sentences describing the nature of the problem.
  • Primary Action – A paragraph or two that provides instruction on the actions to take to try and rectify this problem.
  • Additional Actions – Provides further information for the operator. Usually includes other possible courses of action if the primary action did not produce the desired result.

The following problems (and many others) are covered:

  • Baggage Jam
  • Motor Disconnect
  • Emergency Stop
  • Motor Overload
  • Shaft Encoder Fault
  • PLC Communications Fault
  • Database/Graphics Server Communications Fault
  • Missing Bag Jam

The text that is displayed for problem resolution can be configured by an Administrator in the Web Client. For details, see the HMI Management section in the SmartSort System Administration Manual.
