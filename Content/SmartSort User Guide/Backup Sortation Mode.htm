Backup Sortation Mode

The MEC has a connection to the PLC which allows it to continue to sort bags when the SAC server is down or unavailable as a contingency. If the communication to the SAC is disrupted, the MEC is considered to be in Backup Sortation Mode because direct access to the flight schedule is not available. While in Backup Sortation Mode, an operator has limited sorting capability.

The MEC is connected to the PLC network and to the SAC over Ethernet. Under normal operations, the MEC sends the bag's information to the SAC and the SAC determines the bag's sort destination. However, the MEC can continue to sort bags in the event that the SAC is unavailable. To do so, the MEC Application downloads some data from the SAC at periodic intervals. The following data is downloaded from the SAC throughout the day, typically every 60 seconds:

  • Flight schedule
  • Tag priority information
  • Virtual to physical mapping information
  • Carrier information (default piers)
  • MEC configuration (so the MEC can start up in Backup Mode)

If sorting transactions are made by the MEC while the MEC is in Backup Mode, the statistics about the type of encoding methods used are transferred to the SAC once the MEC is reconnected with the SAC. The actual transaction data is also copied to the SAC once the connection has been re-established. Any data that has not been copied to the SAC is lost if the application is closed before the connection returns.

All Encoding Methods are available in Backup Sortation Mode. For information about encoding methods, refer to section 5.1 Encoding Bags. If a tag is scanned while the MEC is in Backup Sortation Mode, the bag will be automatically released to the associated Carrier Make-Up Unit (if one is assigned). It will not be sorted by flight because no BSM data is available in Backup Mode. If no carrier default pier has been assigned, the bag needs to be encoded using the Pier encoding method (see section 5.2.4 Make-Up Unit Mode), the quick release selection encoding method (see section 5.2.5 Quick Release), or the Flight encoding method (see section 5.2.3 Flight Mode).
