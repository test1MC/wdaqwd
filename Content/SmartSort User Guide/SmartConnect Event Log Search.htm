SmartConnect Event Log Search

The SmartConnect Event Log Search allows the operator to search through different SmartConnect events, such as connection failures or disconnects. The search results display the service event that occurred, and if it was a sending or receiving issue.

The search is generated based on a time range (either fixed or between the current time and a certain number of hours before the current time). If the time range is too broad, only the most recent 5000 logs will be returned. The search can be conducted for a maximum search window of 24 hours.

Time Range criteria are used for filtering the set of results returned. Filters can be applied to each column of the SmartConnect Event Log Search. Search results can also be exported to a CSV file format by clicking Export to CSV.

SmartConnect Event Log Search Results

The following information is included in the Event Log Search:

Column

Description
Timestamp The time when the event occurred (local for the station).
Event Type Type of event: Connection, Disconnection, Restart, etc.
Service The specific service that recorded the event (rarely is more than a single Message Broker Service installed).
Instance The specific instance of the service that recorded the event (rarely is more than a single Message Broker Service or instance installed).
Pipeline The pipelined that recorded the event (usually bag and flight messages are split into separate pipelines).
Agent The specific agent that recorded the event.
Message The details of the event.
