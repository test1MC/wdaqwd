Encoding Bags

Bags can be encoded using several different methods. The following sort methods are available:

  • Scanned Using Hand Scanner – The bag tag can be scanned using the hand scanner. This is the preferred method for encoding a bag. For more information, see section 5.2.1 Bag Tag Scanned.
  • Manually Keyed Bag Tag – The bag tag number can be manually keyed using the on-screen keypad if the hand scanner cannot read the bag tag. For more information, see section 5.2.2 Manually Keyed Bag Tag.
  • Carrier Pier – Allows the operator to send a bag to the default pier for a particular carrier. For more information, see section 5.2.6 Carrier Make-Up Unit Mode.
  • Flight Mode – Allows the operator to view a list of flights and to select the correct flight from the list. For more information, see section 5.2.3 Flight Mode.
  • Pier Mode – Allows the operator to view a list of all of the flights sorted by pier and to select the correct flight from the list. For more information, see section 5.2.4 Make-Up Unit Mode.
  • Quick Release Selection – Allows the operator to quickly release a bag to a pre-configured virtual pier. For more information, see section 5.2.5 Hot Make-Up Unit Selection.

The bag transaction data will not be associated with a bag tag if a bag is not scanned and read by the hand scanner or manually keyed by the user. For example, if a user chooses to encode a bag to a carrier's default pier without scanning the bag tag or entering the number manually, there will be no record of that bag sorting to that destination. Without scanning the bag tag or keying the bag tag number there is no way for the system to associate the bag tag number or the BSM for a bag with the bag that is dispatched from the MEC.

If a bag tag is scanned or manually keyed in and a BSM exists for the bag, and the carrier has BPM sending enabled, a BPM will be sent from the MEC with information about the baggage transaction. Note that Pseudo BSM creation may be configured to send additional BPMs (for information about Pseudo BSMs, refer to section 5.2 Pseudo BSMs).
