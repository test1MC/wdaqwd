SmartSuite Login and Security

A valid User Name and Password are required to use the Web Client application. Anyone that needs to have access to this application is assigned a unique user ID and password.

Each user is also assigned to one or more user groups. User groups are used to categorize users who perform similar functions (e.g. supervisors, operators, etc.). Each user group has access to particular screens and functions. These permissions can be adjusted at any time (typically by an administrative user). Any changes made to a user group will affect all members of that group.

Once the login is successful, the user will have access to any screens and functions for which they have been granted permission. If the user is inactive for a prescribed amount of time, the session is terminated and the user will need to log back in.

Warning: If changes are made to a form on the SmartSuite Enterprise Web Client, those changes must be saved by clicking on the Save button or icon. Clicking on a link to another page or closing the web browser without saving the changes may result in those changes being lost.

Signing In

When the Web Client is first accessed, the Welcome screen is displayed with the login fields.

Figure 2‑4: Welcome Screen - Login Fields

To log into the web client:

  1. Enter the User ID in the Login field.
  1. Enter the password in the Password field.
  1. Click Login to sign into the Web Client.

Signing Out

To sign out of the Web Client, select Sign Out from the Main Menu. Logging off is not necessary; users will be redirected to the Sign In screen if their session has expired due to inactivity.
