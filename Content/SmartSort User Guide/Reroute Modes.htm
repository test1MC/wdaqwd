Reroute Modes

The flights assigned to each Reroute group can be dynamically rerouted in one of three ways. The reroute method is selected from the Reroute Mode drop-down list. The available reroute modes are:

  • Immediate Destination – Looks for a flight whose first flight leg matches the bag's immediate destination.
  • Direct Final Destination – Looks for a flight whose first flight leg is the same as the bag's final destination.
  • Indirect Final Destination – Looks for a flight that has any leg that is the same as the bag's final destination.

Bags can be rerouted in one of four ways:

  • Dynamic Reroute – Dynamically looks for the next available flight using the configured rules and sends the bag to the Make-Up Unit for that flight.
  • Flight Missed Pier – Sends the bag to the Missed Pier defined for the original flight or the Global Missed Pier if it is mapped.
  • Group Default Pier – Sends the bag to the sort destination configured for the reroute group.
  • Regular Pier – Sends the bag to the time-based pier based on the regular sortation parameters (if the bag has missed its flight, it will be sent to the "Missed Pier"). This reroute options is essentially the same as not using reroute.

If the configured reroute method cannot be used to reroute a bag (for example, if a dynamic reroute flight cannot be found), the system will attempt to reroute the bag using the other reroute options. The sequence in which the reroute options will be attempted is listed below.

If a flight is configured for Dynamic Reroute, and a reroute destination cannot be determined using the configured reroute method, the system will try to reroute the bag using the next reroute method in the sequence below. The system will continue to attempt reroute methods until it is able to successfully reroute the bag, or until the last method in the sequence is reached.

For missed bags

  1. Dynamic Reroute
  1. Reroute Group Pier
  1. Regular Pier

For cancelled bags

  1. Dynamic Reroute
  1. Flight Missed Pier
  1. Reroute Group Pier
  1. Regular Pier

For early bags (no cascading logic between items 1 and 2)

  1. Dynamic Reroute
  1. Reroute Group Pier
  1. Regular Pier

For example, with a bag on a cancelled flight, if dynamic reroute is configured and no next available flight can be found, the system will attempt to send the bag to the original flight's missed pier. If no missed pier is defined for the flight, the system will attempt to send the bag to the default reroute pier for the group. If the default reroute pier is not defined, the system will attempt to sort the bag normally (in this case to the On Time sort destination).
