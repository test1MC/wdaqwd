Reroute Search Window

The Reroute logic looks for matching reroute flights within a certain time window (so that bags are not held indefinitely waiting for a reroute flight). This is called the "Search Window". The search window determines which flights will be considered for reroute by looking for matching flights with departure times that fall within the search window.

Search Window

The Search Window is configured using two fields:

  • Search Window Start Offset – Represents the start of the search window. This is the number of hours and minutes after the current time to search for another flight. By default, the start offset time is set to 30 minutes. The offset is recommended because once a bag is assigned to a new flight, a certain amount of time is required to sort the bag to the correct sort destination. If a bag is assigned to a flight that departs in two minutes, the bag would likely miss that flight and would be sent back for reroute.
  • Search Window End Offset – Represents the end of the search window. This is the number of hours and minutes after the current time to search for another flight. By default, the end offset time is set to eight hours. This offset is recommended to make sure bags are not held indefinitely waiting for a reroute candidate.

The following example illustrates how the search window works during a typical reroute scenario. If a bag has missed its flight and is re-scanned in the BHS at 14:00 with the reroute search window configured as 30 minutes to five hours, the system will look for suitable flights with departure times between 14:30 and 19:00.

Reroute Day

Since flights do not necessarily depart 24 hours a day, the system also uses a "Reroute Day" to further define when the system will look for reroute flights. The Reroute Day is defined using two fields:

  • Start Time – The earliest time (in 24h format) that the system will normally be able to search for reroute candidates. This is typically the earliest time that flights are scheduled to depart at the associated facility. If flights are regularly scheduled to depart 24 hours a day, defining a Reroute Day is not required.
  • End Time – The latest time (in 24h format) that the system will normally be able to search for reroute candidates. This is typically the latest time that flights are scheduled to depart at the associated facility. If flights are regularly scheduled to depart 24 hours a day, defining a Reroute Day is not required.

If a bag misses its flight or a flight is cancelled after the time entered in the End Time field, the system will first look within the search window as described previously. However, if no available flight is found within that window, the system will look for a flight starting at the beginning of the next reroute day (the time entered in the Start Time field) and continuing for the number of hours specified by the reroute search window, provided that the Allow Search Next Day option is selected.

The following example is meant to clarify how the "Reroute Day" impacts the search window. The example assumes the following settings.

  • Start Offset of 30 minutes
  • End Offset of five hours
  • Start of Reroute Day set at 07:00
  • End of Reroute Day set at 22:00

If a bag missed its flight and is scanned at 20:30, the system will first look for all flights departing from 21:00 to 01:30 (the search window is based on the 30 minute Start Offset and the five hour End Offset). If no available flight is found in that window, and the Allow Search Next Day checkbox is selected, the system searches for flights within the reroute window starting at 07:00 the next day. If the End Offset is configured for five hours, the system looks for flights with departure times between 07:00 and 12:00 (the five-hour End Offset).

If the Allow Search Next Day field is not selected, the reroute logic will only look for flights from 21:00 until 01:30. In this case, the system would attempt the next reroute method (after Dynamic Reroute). See section 4.4.1 Reroute Modes for details about the different reroute methods and the sequence in which each method will be used.
