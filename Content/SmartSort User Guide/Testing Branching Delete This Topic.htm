The Activity Log Search screen allows the operator to search a less detailed version of the log files generated by the various services/applications from within the Web Client. The Activity Log itself is described in section 7.4.2.2 Activity Log. Note that the time zone associated with the timestamp returned in the search results is indicated in the top right corner of the screen.

The following search criteria is available if the applicable data is received from the data interfaces:

  • Application (Is In / Not In) – Indicates the service associated with the returned activity log entry.
  • Device (Is In / Not In) – Indicates the device associated with the returned activity log entry.
  • Entry (Is In / Not In) – Indicates the activity log entry type.
  • Message (Contains / Is / Is Not / Begins With / Ends With) – Indicates which keywords should be used/omitted in order to refine the search results.
  • Message Type (Is In / Not In) – Indicates the message type associated with the returned activity log entry.
  • Severity (Is In / Not In) – Indicates the severity (Error, Information, or Warning) associated with the returned activity log entry.
  • Subsystem (Is In / Not In) – Indicates the subsystem associated with the returned activity log entry.
  • Time Range (Current / Fixed) – The timeframe used to locate a specific activity log entry.
  • User (Is In / Not In) – Indicates the User associated with a specific activity log entry.
  • Value (Is In / Not In) – Indicates the numerical value associated with the returned activity log entry.
