Controlling the System Using the HMI

The HMI provides control functionality for a number of system devices for users with sufficient privileges. This control functionality is described in the following sections.

On startup, the HMI application is automatically logged in to the "User" account, which has read-only privileges. In order for the HMI to perform any control functions, it must be logged into with a higher privileged user account. Default security for the HMI application is tied to the current Windows Active Directory user, but this can be changed by logging in with a different user account. The following table defines the HMI security levels that are available.

HMI Access Level

Description

Read-Only User

Ability to view all screens within the application. No control functions available.

Operator

Access to most control functions (e.g. Push/Divert All Push/Divert Half).

Administrator/Engineer

Access to all functions

 

The HMI application includes a means of logging in directly with one of the HMI access levels. Selecting Login from the Utilities menu will display the Login window, which allows the user to change accounts with the appropriate credentials.
