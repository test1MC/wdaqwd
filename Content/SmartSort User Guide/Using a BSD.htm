Using a BSD

When a BSD station first starts, the BSD screen displays "Station Disabled." Until a user enables the BSD station, the BSD station remains disabled. If a bag stops at the queue belt for the disabled BSD station, the status of the bag displays, but no action can be taken until a user enables the BSD station.

Figure 60: BSD - Station Disabled Screen

Once a user taps the Enable button, the BSD is considered enabled and the user can begin processing bags.

When a bag enters CBRA, the bag is conveyed to the furthest downstream station available on the alarm line. When a BSD station is available, it will indicate a "Waiting for Bag" state using an on-screen message.

Figure 61: BSD - Waiting for Bag Screen

When a bag arrives and stops at a BSD station, the state of the BSD station changes depending on the status of the bag, and the bag information along with the Bag Security Status displays on the BSD screen.

The bag information that displays on the BSD main screen includes:

  • Bag Status ‑ The status us of the bag (for example, Alarmed, Cleared, etc.).
  • Bag ID
  • ISD ID - The ISD machine name or the screening line at which the bag was processed.
  • Found at PE - The location where the system began tracking the bag. This information is only displayed for bags in the BHS Unknown state.

The Bag Security Status is indicated with a color associated with the security status. These colors and statuses include.

  • Alarmed (Red) - The bag was not cleared by the ISD machine and/or OSR.
  • Cleared (Green) - The bag was cleared by the ISD machine or OSR. Bags should rarely arrive in CBRA with a Cleared status.
  • Out of Gauge (Orange) - The bag was too large to go through the ISD machine.
  • ISD Error (Yellow) - The bag could not be properly processed by the ISD machine.
  • BHS Unknown (Blue) - The bag was lost in tracking after the ISD machine.
  • Timeout (Brown) - The bag required OSR and no decision was made in the time allotted.
  • Communication Error (Black) - The bag did not receive a decision from the ISD machine.

Depending on the Bag Security Status, the BSD displays different processing options. The processes for each security status type are outlined below.

To process an Alarmed bag: 

  1. Tap the red Inspect button. The Bag ID is transferred to the Secondary Viewing Station (SVS) and the BPT is started.
  2. Manually remove the bag from the queue conveyor and transfer it to the BIT. The BSD screen displays Clear and Resend buttons.
  3. If the Bag ID was not transferred successfully to the SVS, press Resend to try transferring the information again.
  4. Inspect the bag, and then do one of the following:
  • If the bag is clear, place the bag on the clear line and then press the Clear button on the BSD screen. The BPT stops and the bag is reinserted into the BHS.
  • If the bag is not clear, follow Standard Operating Procedures (SOP).

To process a Cleared bag: 

  1. Tap the green Remove button. The Bag ID is transferred to the SVS and the BPT is started.
  2. Manually remove the bag from the queue conveyor and transfer it to the BIT. The BSD screen displays Clear and Resend buttons.
  3. If the Bag ID was not transferred successfully to the SVS, tap Resend to try transferring the information again.
  4. Place the bag on the clear line and then tap the Clear button on the BSD screen. The BPT stops and the bag is reinserted into the BHS.

To process an Out of Gauge (OOG) bag: 

  1. To inspect the bag, tap the orange Remove & Inspect button. The Bag ID is transferred to the SVS and the BPT is started.
  2. Tapping the Reinsert button will move the bag downstream for automatic reinsertion. The BPT will not start and the BSD station will become available again.
  1. Manually remove the bag from the queue conveyor and transfer it to the BIT. The BSD screen displays Clear and Resend buttons.
  1. If the Bag ID was not transferred successfully to the SVS, tap Resend to try transferring the information again.
  1. Inspect the bag, and then do one of the following:
  • If the Bag ID was not transferred successfully to the SVS, tap Resend to try transferring the information again.
  • If the bag is not clear, follow Standard Operating Procedures (SOP).

To process an ISD Error bag: 

  1. To inspect the bag, tap the red Inspect button. The Bag ID is transferred to the SVS and the BPT is started.
  2. Tapping the Reinsert button will move the bag downstream for automatic reinsertion. The BPT will not start and the BSD station will become available again.
  1. Manually remove the bag from the queue conveyor and transfer it to the BIT. The BSD screen displays Clear and Resend buttons.
  1. If the Bag ID was not transferred successfully to the SVS, tap Resend to try transferring the information again.
  1. Inspect the bag, and then do one of the following:
  • If the bag is clear, place the bag on the clear line and then tap the Clear button on the BSD screen. The BPT stops and the bag is reinserted into the BHS.
  • If the bag is not clear, follow Standard Operating Procedures (SOP).

To process a BHS Unknown bag: 

If the photo eye was accidentally flagged and no bag is present at the queue conveyor, tap the orange Reset BSD button. The BSD station is reset and indicates a "Waiting for Bag" state.

Figure 62: BSD - BHS Unknown Bag

If a bag is present at the queue conveyor, scan the bag tag using the hand scanner.

Note that moving the bag to access the tag will not cause the conveyor to move, nor will it trigger any events or alarms. When a tag is scanned, one of the following will occur:

  1. If information is available for the bag, the BSD screen displays the applicable processing buttons according to the bag status. Follow the appropriate process.
  2. If no information is available for the bag, tap the purple Failed Scan button. The BSD screen displays Reinsert and Inspect buttons.
  3. Scanning other tags on the bag will automatically refresh the screen if there is a successful scan. The applicable processing buttons display according to the bag status.
  4. Tapping Reinsert will move the bag downstream for automatic reinsertion.
  5. Tapping Inspect will process the bag as an Alarmed bag (see process above).
