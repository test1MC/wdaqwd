Grid Basics

This section describes how to interact with the grid in the Editor view. The following functions are available:

  • Copy/Paste – Each cell/selection can be copy and pasted with the keyboard using CTRL+C and CTRL+V respectively. Alternately, if copying cells in rows adjacent to one another, click the cell that contains the data that should be copied and then grab the handle on the bottom right corner of the cell and drag down across any cells that should have the same value.
  • Multiple Selection – Multiple rows/cells can be selected at a time to perform a command on (described later) or copy from. Multiple selections can be achieved in one of two ways:
  • Click any cell, hold the Shift key, and then click on another cell. All the cells in between the two clicked cells will be highlighted. For example, if you click on a cell in the first row of the first column and then hold the Shift key and click on the seventh row in the third column, all the cells from column 1 to 3 and row 1 to 7 will be highlighted/selected.
  • Click any cell, keep the mouse button held down, and drag the mouse to any other cell, and then release the mouse button. All the cells in between will be highlighted/selected.
  • Click a row number to select an entire row or click a column header to select an entire column.
  • Inline Editing – Each cell can be manually edited. Some cells contain drop-down lists (e.g. Airline). Note that date and time fields must be entered in the format of yyyy-mm-dd hh:mm (e.g. 2015-12-25 01:30). The time must be entered in 24:00 format.
  • Pier Editing – A single pier can be assigned for all bags for a flight by typing the pier name. Optionally a station and/or class can be added to a pier assignment. To configure the pier based on station and/or class, the pier information must be formatted as follows (with commas separating additional pier assignments):
  • PierName/Station/Class
  • PierName/Station/Class, PierName/Station/Class, etc.
  • An example complicated pier assignment might look like the following:

  • Adding Flights – There is always an empty row at the bottom of the grid that can be used to add a new flight. One or more flights can also be pasted to the last row or duplicated from other existing flights. Lastly, the Add Flight button can be used to add a new flight (see section 4.3.3.2.7 Adding a Flight to the Flight Schedule Template). Note that the flight will not actually be added to the flight schedule template until the Save All button is pressed.
  • Deleting Flights – One or more flights can be deleted by selecting the entire row and hitting the delete key. The entire row is selected by clicking the row number on the left side of the grid. The Delete button can also be used to delete a flight (see section 4.3.3.2.10 Deleting a Flight from the Flight Schedule Template). Note that the flight will not actually be deleted from the flight schedule template until the Save All button is pressed.
  • Editing Flights – Individual flights can be edited by modifying the text in the grid or by clicking on the Flight Number in the grid, which will display the Edit Flight dialog. The Edit Flight dialog can also be displayed by clicking the Edit Flight button (see section 4.3.3.2.8 Editing a Flight in the Flight Schedule Template). Note that the flight will not actually be edited in the flight schedule template until the Save All button is pressed.
  • Duplicating Flights – One or more rows can be duplicated by selecting the entire row and clicking the Duplicate button. All of the flight details, including the flight number, are copied when a flight is duplicated. Duplicated flights will be added to the grid with a status of NEW until changes are saved.
  • Row and Cell Validation – When data is entered or edits are made, rows and cells are validated. If there is an issue with a cell, the cell is highlighted in red and an error message is displayed in the Status column. Rows cannot be saved when errors are present - if the Save All button is clicked any other changes will be saved, but not the invalid rows. If there is a flight without a proper On Time Pier assignment, the On Time Piers cell is highlighted in yellow, however flights can be saved without an On Time Pier assignment.
  • Status Column – The last column of the grid is a status column. This has the following values/states:
  • DELETE – Gray Text – The row has been marked for deletion but not yet saved (actually deleted)
  • INVALID – Highlighted Red – There is a problem with one or more cells in the row (e.g. the open time is after the close time, the Open time has not been entered, etc.).
  • NEW – Gray Text – A new row has been added, but not yet saved.
  • MODIFIED – Gray Text – The row has been modified in some way, but not yet saved.
  • SAVED – Green Text – The row has been saved to the server.
  • OFFSET INVALID – Highlighted Red – There is a problem with the Open, Late or Closed time.
  • Shifting Flight Times – The Scheduled Departure flight time can be shifted for any selected rows. If multiple flights are selected, the specific flights that are affected can be displayed or hidden. Changed flights will be updated in the grid with a MODIFIED status until changes are saved.
  • Sending Flights to the Active Flight Schedule – One or more rows can be sent to the active flight schedule.
  • If the same flight already exists in the active flight schedule, it will be overwritten by the new flight.
  • Flights will be copied to the active flight schedule regardless of the effective and discontinue dates as well as the days of the week the flight operates on.
  • If there are multiple flights with the same flight number selected, only the last flight in the selection will be copied to the active flight schedule.

None of the actions performed in the grid or commands performed will be saved to the server until Save All is selected. Flights with unsaved changes cannot be sent to the active flight schedule.

Note that any rows with a Status of INVALID will not be saved until the row is corrected. Once the save is complete, the Status of all the saved rows will change to SAVED.
