Reroute Control

During the sortation process, bags that have missed their flights or bags for a flight that has been cancelled can be automatically redirected to the next available flight. All flights belonging to the same reroute group can be used as candidates when attempting to find a next available flight.

A default "No-Reroute" group is set up which is the default reroute group for all flights. The "No-Reroute" group cannot be modified, but additional reroute groups can be configured.

To access the Reroute Groups screen, click the Main Menu icon (), select Sortation from the available options, then click Reroute Groups.

Figure 4‑35: Reroute Groups Screen

The Groups Details section of the screen is divided into four main sections:

  • Identification – This section contains the reroute group name as well as the general reroute mode. The general reroute mode determines which types of flights will be considered candidates for reroute (direct flights, flights with multiple legs, etc.). For more information about the available reroute modes, refer to section 4.4.1 Reroute Modes.
  • Time Settings – The time settings determine the search window during which the system will attempt to find a reroute flight. For more information, please refer to section 4.4.2 Reroute Search Window.
  • Flight Options – The flight options configure whether or not a bag can be rerouted if its flight is currently on hold. The system can also be configured to reroute a bag to a flight with an earlier scheduled departure time. For more information, please refer to section 4.4.3 Flight Options.
  • Reroute Type Settings – The additional Reroute settings is used to configure how bags will be rerouted in four different reroute scenarios. For additional information, refer to section 4.4.4 Additional Reroute Settings.
