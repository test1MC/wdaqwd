Monitoring Alarms with the HMI

Alarm Monitoring and History

The HMI Application provides Alarm Monitoring capabilities. Current Alarms can be acknowledged by the operator that will stop the alarms from flashing or making noise, but alarms will not be cleared from the screen until the fault has been reset in the field. Historical alarms can be searched within the SmartSuite Web Client (see section 10.3.3 Alarm History Search).

Click on the Alarm Summary button on the Title/Navigation Bar to obtain a summary of all alarms that have not been cleared and acknowledged.

Figure 7‑22: Alarm Summary Screen Sample

The following controls will be available on the Alarm Summary screen:

Control

Result

Ack Current

Acknowledges the currently selected alarm.

Ack Page

Acknowledges all alarms currently displayed in the Alarm Summary window (the filtered list of alarms).

Ack All

Acknowledges all alarms in the system.

Silence All

Silences all alarms in the system (without acknowledging them).

Pr. Resolution

Displays the Problem Resolution Text window.

Alarm Sound

Displays the Alarm Sound Status pop-up window:

Restart Alarms

Restarts the Alarm Manager

Restart Events

Restarts events (a system maintenance function)

Alarm History

Displays the Alarm History screen
