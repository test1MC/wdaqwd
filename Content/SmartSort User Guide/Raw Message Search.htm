Raw Message Search

The Raw Message Search screen allows the operator to search through the strings of raw data that are received from the Airport Host computers if the applicable data is received from the data interfaces. This data is primarily for BSM and BPM Messages, but could include other types of messages as well. Note that the time zone associated with the timestamp returned in the search results is indicated in the top right corner of the screen.

The following search criteria is available if the applicable data is received from the data interfaces:

  • Comment (Contains / Is / Is Not / Begins With / Ends With) – Indicates which keywords should be used/ommitted in order to refine the search results.
  • Message Text (Contains / Is / Is Not / Begins With / Ends With) – Indicates which keywords should be used/ommitted in order to refine the search results.
  • Source (Is In / Not In) – Indicates the source or origin of the associated raw message (e.g. BSM Connection Name).
  • State (Is In / Not In) – Indicates whether the associated raw message was sent or received by the system.
  • State Timestamp (Current / Fixed Time Range) – The time that the associated raw message was sent or received by the system.
  • Timestamp (Current / Fixed Time Range) – Specifies the range of time during which the associated raw message was recorded by the system.
  • Type (Is In / Not In) – Indicates the associated message type of the returned raw message result.
  • Valid (Is In / Not In) – Indicates whether or not the returned raw message is valid.
