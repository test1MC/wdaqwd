PRTG Monitor

Each time the PRTG software (see section 3.3.3.3 Server Hardware Monitoring) detects a change of state, it logs that change in a database table. The PRTG Monitor service is responsible for picking up this logged data and acting on it to set the alarms on the HMI appropriately and create new alarm entries on the system dashboard.

The service also resynchronizes all PRTG sensors each hour in case one of the sensor changes is missed.

This service must remain operational for the Watchdog/Monitoring alarms on the HMI to be functional. If it is not running, server and other hardware status would have to be monitored manually for the relevant third-party applications.
