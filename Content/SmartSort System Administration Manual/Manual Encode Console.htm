MEC Releases Bags from Wrong Encoding Location

MEC lines have been temporarily switched:

  1. Ensure that the MEC name in the upper left corner of the screen is correct.
  1. Use the Switch Lines key-switch button to move the MEC back to its appropriate location.
MEC Appears Stuck or Frozen

Slow PLC connections or disconnected PLC:

  • If the MEC is stuck releasing a bag for more than 15 seconds, there may be a communications problem.

MEC Operating System (Windows) has frozen:

  • The MEC application is alive if the clock in the lower right corner shows accurate time. If the time is not changing, the application or computer needs to be restarted.
PLC Connection Icon is Red and Bags Cannot Be Released

Ethernet cable is not connected to the MEC:

  1. Check the BHS switch and MEC network cables.
  1. Confirm that the MEC computer can be reached via ping from another computer.

PLC Ethernet connection issue:

  1. Check that the PLC Ethernet card is running correctly and can be accessed/pinged.
  1. Check the PLC to ensure that the PLC is running.
Server Connection is Red ("Backup Mode")

The Sort Allocation Controller (SAC) is disconnected from the MEC:

  1. Verify Sort Allocation Controller Status on HMI.
  1. Ensure bags are sorting automatically at the ATR.
  1. Check Ethernet cables/switches at the server end.
Can't Release Bag at MEC Scan

Error occurred during automatic sortation:

  1. Check the Action Log on the right side of the MEC screen to see what the most recent error was (at the bottom of the list).
  1. The error should be self-explanatory (e.g. Flight Not In Schedule, or Flight Makeup Unit not Assigned) and must be corrected by operations personnel.
Error Message "BSM Not Found"

Scanning an old or invalid barcode:

  1. There must be a valid barcode on the bag that was recently printed.
  1. Attempt to release bag with Flight or Makeup Unit Sort mode.

Barcode quality is poor:

  • Try to type in the BSM number on the keypad to sort automatically.

No connection to SAC:

  • Is the Status Light Green, displaying "All Connections OK"?

Not receiving sufficient BSMs from Carrier Host

  • Verify BSM connection status on HMI.
  • Search for bag tag from Global Search in the Web Client to see if a BSM for the bag was received.
"Release to Carrier Default Makeup Unit" Dialog Does Not Appear for Some Bags

This dialog is only present if the operator can sort by Carrier.

The Carrier Default Makeup unit must be enabled in the Web Client Application:

  • In the Web Client, verify that Uses Default Virtual Makeup unit is checked for that Carrier in the Location Settings 2.2.3 Location Settings screen (see section 2.2.3 Location Configuration).
