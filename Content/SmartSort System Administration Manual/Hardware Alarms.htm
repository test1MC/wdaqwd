Hardware Alarms

The following sections outline outlines the common hardware alarms, and the probable causes and remedies for those alarms.

Hardware Alarm

Server monitoring software has detected a problem on a physical server:

  • Check the Upper Level Alarms screen in the SmartSuite Enterprise Web Client to determine which server (Host) is faulted.
  • Check the LEDs or status display on the front server panel to identify quick status updates. Please refer to the Dell Manual for information on status LEDs.
  • Check PRTG to determine if there is a specific hardware fault withing the physical server (e.g. temperature fault).
  • Contact Brock Solutions for support.
SmartSort Alarm

Server monitoring software has detected SmartSort or other critical BHS service is not running:

  • Check the Upper Level Alarms screen in the SmartSuite Enterprise Web Client to determine which service is not running.
  • Check the SmartSuite Service Manager on all servers to find out which service is no longer running and restart it.
  • If the issue is not resolved by a service restart, check the service log files for any errors.
  • Contact Brock Solutions for support.
Virtual Machine Alarm

Server monitoring software has detected a problem with a virtual machine

  • Check the Upper Level Alarms screen in the SmartSuite Enterprise Web Client to determine which virtual machine is not running.
  • Open vSphere and turn on/restart the virtual machine is not running.
  • Contact Brock Solutions for support.
Network Alarm

Server monitoring software has detected a problem on a switch:

  • Check the Upper Level Alarms screen in the SmartSuite Enterprise Web Client to determine which switch is faulted.
  • Check that all system switches are powered up correctly and operating as expected.
  • Check PRTG to determine if there is a specific fault within the switch (e.g. temperature fault).
  • Contact Brock Solutions for support.
Resources Alarm

Server monitoring software has detected the amount of free hard drive space, CPU, or RAM on the servers has dropped below the configured threshold:

  • Check the Upper Level Alarms screen in the SmartSuite Enterprise Web Client to determine which virtual machine is affected. Once the affected VM has been determined, you will be able to determine what specific issue triggered the alarm (e.g. low on hard drive space, CPU, or RAM).
  • If the issue pertains to hard drive space, identify which files/folders are taking up the most space and consult Brock Solutions before taking additional action.
  • If the issue pertains to the CPU, identify which processes are using high amounts of the CPU using Task Manager and consult Brock Solutions before taking additional action.
  • If the issue pertains to RAM, identify which processes are using high amounts of memory using Task Manager and consult Brock Solutions before taking additional action.
Ping Alarm

Server monitoring software has detected a network device that is no longer reporting its status to the BHS:

  • Check the Upper Level Alarms screen in the SmartSuite Enterprise Web Client to determine which device on the network is no longer responding to the PING call.
  • Check to see if the device is powered on and has a network connection.
  • Contact Brock Solutions for support.
SAN Alarm

Server monitoring software has detected a problem on a SAN:

  • Check the Upper Level Alarms screen in the SmartSuite Enterprise Web Client to determine which device on the SAN is faulted.
  • Check the LEDs or status display on the front server panel to identify quick status updates. Please refer to the Dell Manual for information on status LEDs
  • Check PRTG to determine if there is a specific fault within the SAN (e.g. temperature fault).
  • Contact Brock Solutions for support.
