If replication fails, the distributor provides details of what went wrong. Use SQL Management Studio to correct the error or re-initialize the publication:

  1. Open Replication Monitor.
  1. Expand Publishers.
  1. Expand the SAC entry.
  1. Click on the Publication.

Finding out what went wrong:

  • The right pane of the display shows one line for each replication process. The process that failed is indicated, along with a brief description of the problem.
  • Right-click the failed process and select View Details to see a new window with more information about the current replication status at the subscriber.

Correcting the Replication Problem

If the error details contain enough information, correct the problem by manually updating the appropriate tables. Some possible error examples include:

  • A row was missing on the DHS Subscriber.
  • A new row could not be added to the Subscriber because it conflicted with an existing row.

The problem can be corrected by inserting the missing row or deleting the duplicate row on the DHS.

If the problem can be corrected manually, the Replication process will start up automatically. (Use Start Synchronizing on the subscription context menu if necessary.)

If the error details are not clear, most problems can be solved by reinitializing the replicated database as described in 3.3.9.3 Reinitializing the Publicationthe next section.

Reinitializing causes the SAC to create a snapshot of its current data. All corresponding data on the DHS is deleted, and replaced with data from the SAC.

Reinitializing can take some time (around 30 minutes for full synchronization). The database should only be reinitialized if the problem cannot be easily corrected. When the Reinitialization process occurs, it locks the database tables and prevents access by the SmartSort services. For this reason, Reinitialization should not be performed during live operations.
