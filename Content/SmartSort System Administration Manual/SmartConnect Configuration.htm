SmartConnect ConfigurationMessage Broker Service

The SmartConnect Message Broker is a background process that provides connectivity between SmartSuite and external systems through the exchange of messages.

The message broker configuration is organized into pipelines, with each pipeline representing an independent flow of messages. Typically, one pipeline is configured for baggage messaging (BSM, BPMs, etc.) and one pipeline is configured for flight messaging.

Components called “Agents” within the message broker are used to perform a variety of tasks such as filtering unneeded messages, routing a message to another connection and processing the message for storage within the SmartSuite system.  These agents work together to perform the overall functions of:

  • Creating bag and flight records in the database
  • Queuing messages when external systems are unavailable

Communication Protocols

The message broker supports a wide variety of protocols to communicate with external systems.  This includes the following that are setup for DFW-D:

  • Sabre
  • WebSphere MQ Client
  • ARINC BagLink
  • File Input and Output
  • Database Query (MSSQL, ODBC, etc.)
  • Raw TCP stream

Message Formats

The message broker is capable of receiving any format of message data and forwarding it to another system.

In order to process messages into bag and flight records in the database, the message must be one of the supported formats.  These include:

  • For bag records:
  • IATA Baggage Information Messages as defined in Recommended Practice 1745 (BSM, BPM, BUM, etc.)
  • For flight records:
  • IATA AIDX flight data messages for the DFW AODB Connection
  • AA BOSS flight data messages for the AA BOSS Connection
