VMware High Availability Solution

VMware is used to achieve system redundancy. This software provides a completely virtualized set of hardware to the server operating system. In this way, VMware virtual machines become highly portable between computers, because every physical server looks nearly identical to the application server. Additionally, the high availability components of VMware allow for the automatic migration of operational VMs, in the event of a failure (see section 3.1.2.1.4 VM Migration). This is done in a way that is seamless to the end-user, although there will be a brief interruption during migration to the alternate host during which the applications may not be available. However, no manual intervention is required to complete this process.

A Storage Area Network (SAN) is used so that the data is available to both servers. The hosts do not store any data locally other than the VMware operating system. This architecture is highlighted in the diagram below.

Figure 3‑3: VMware Architecture

As indicated in the diagram above, the VMware architecture supports multiple Virtual Machines (VMs) within the same architecture. Within VMware, the VMs are configured as VMware HA (High Availability) VMs. With HA Protected VMs, the VM runs on one of the two VMware hosts, and upon failure of the host, the VMs running on that host are failed over on the alternate hosts. In this case, all VMs running on the host that failed get migrated over to the alternate host.

The typical components for a VMware HA solution are two hosts, the latest model of mid-sized servers, an entry-level SAN, and two stacked Cisco switches. Brock Solutions typically uses HP or Dell components for a VMware HA solution. The solution uses RAID data storage consisting of two arrays.
