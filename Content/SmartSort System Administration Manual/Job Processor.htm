Job Processor

Data is transferred from the SAC database to the DHS by three mechanisms: SQL Replication (described fully in section 3.4.8), data transfer by ETL (see section 3.4.5.7), and directly by the Job Processor. Some components of the system create events, which are recorded in the SAC database, and are then read by the Job Processor to create or update records on the DHS database.

The Job Processor Service must continue to run for the information in the DHS database to remain up to date. Event records will still be created and queued if the service is not running, but the Jobs need to run their processes to get the data to transfer to the DHS.
