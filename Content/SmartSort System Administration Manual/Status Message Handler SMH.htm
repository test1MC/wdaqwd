Status Message Handler (SMH)

The Status Message Handler receives all status update messages (non-Sort) from the system PLCs. Messages sent to the SMH are used to generate bag transactions that allow the operator to follow bag flow using the Web Client Bag Transaction search or a variety of reports. Types of messages include:

  • Divert Message – this message indicates that a bag was successfully sorted, or failed to sort, or was lost/unknown at a tracking photo eye.
  • ISD Entry/Exit Message – this message is sent for a bag entering an ISD.
  • ISD Result Message – this message is sent when a security decision is bag in screening.

Messages are generally sent by the PLC as bag events occur in the system. The PLC maintains a small queue of messages so that data is not lost during brief outages or loss of connection to the server. Messages are not generally explicitly acknowledged. Each instance of this service communicates to the PLCs over Ethernet, but it is the PLC's responsibility to initiate the connection.
