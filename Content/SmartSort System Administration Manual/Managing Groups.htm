Managing Groups

System Administrators can create and define security groups using the Users and Groups screen. This screen contains lists of all configured groups and users in the system.

Selecting a group from the Groups List populates the Group Details section with information about that group, including a list of the group's members. System Administrators can perform the following actions on this screen:

  • Add a new group
  • Modify an existing group
  • Copy an existing group (does not include copying the users within the group)
  • Delete a group
  • View all the users of a group

The System Administrator can grant particular privileges to groups used by the SmartSuite Web Client. Groups can be permitted access to most of the individual screens or groups of screens.

To add a new group:

  1. Click Add Group in the Groups List section of the screen. The Group Details section displays, allowing the System Administrator to enter details for the new group.
  1. Enter the desired group name.
  1. All groups must exist under another group. To position the new group under an existing group (such as Administrators), use the Add Group Under drop-down list.
  • System Administrators can add groups to the root level by selecting Root.
  1. If members of the group are to be System Administrators (i.e. if they are to have access to all screens and functions), check the System Administrator checkbox. Otherwise, use the checkboxes in the Available Privileges section to select the required access privileges.
  1. To assign users to the group, click the Assign User(s) button.
  • Users that have already been configured are selectable from the Add Users to Group selection list. Multiple users can be added by selecting multiple names. Click Done once all the desired users have been assigned.
  1. Use the checkboxes in the Available Privileges section to select the required access privileges. For example, to grant particular Web Client – SmartSort menu privileges, scroll down the privileges list to select the privileges accordingly. Selecting the Full Access checkbox for a selected privilege type (e.g. Carrier, Web Client, etc.) grants members of the group access to all the privileges for that type.
  1. Click Save at the top of the screen to save the new group.
  • Clicking Cancel or navigating away from the page cancels any unsaved changes.

To modify an existing group:

  1. Select the appropriate group from the Groups List. The group's details display in the Group Details section.
  1. Use the fields, lists, and checkboxes in the Group Details section to modify the group's settings as necessary.
  1. Click Save at the top of the screen to save the changes.
  • Clicking Cancel or navigating away from the page cancels any unsaved changes.

    • To copy an existing group:

    1. Select the group to be copied from the Groups List. The group's details display in the Group Details section.
    1. Click Copy. A new group is created with all the appropriate checkboxes pre-selected. Once copied, the Group Name field will be blank and no users will be assigned to the group.
    1. Enter the desired name for the new group, assign users to the group using the Assign Users button, and if any changes need to be made to the group's privileges, use the checkboxes to make these changes.
    1. Click Save at the top of the screen to save the new group.
  • Clicking Cancel or navigating away from the page cancels any unsaved changes.

    • To delete an existing group:

    1. Select the group to be deleted from the Groups List. The group's details display in the Group Details section.
    1. Click Delete.
    1. Confirm the deletion. A Delete Confirmation message displays. Click Ok to delete the group. A message displays to indicate that the group was successfully deleted.
  • To cancel the Delete operation, click Cancel.
