VM Migration

Note: Users should not need to migrate VMs on a regular basis. VM migration should only be considered if maintenance is being conducted on a host.

The vSphere Web Client can be used to migrate VMs from one host to another. Migrations can be performed while the VM is running; however, there will be a very brief (typically less than 1 second) interruption to network access.

To migrate a VM:

  1. From the vSphere Web Client main screen, browse to Home > VMs and Templates.
  1. Right-click a VM to migrate from the navigation bar.
  1. Click Migrate.
  1. Select a Migration Type.
  1. Note: Only the host should be changed. Do not change the datastore unless otherwise directed by Brock Solutions support.
  1. Click Next.
  1. Select a compute resource.
  1. Click Next.
  1. Confirm the network is mapped correctly on the new host. The Destination and Source networks should match.
  1. Click Next.
  1. Review the migration information. Click Finish. The migration begins.
  1. The progress of the migration displays in the Recent Tasks section of the screen
