Servers

Shutting down or restarting a BHS virtual server should be undertaken with care to avoid affecting live operations, or in conjunction with Brock Solutions support.

For all the virtual servers and the maintenance server, the process for shutting down or restarting is not complex. Simply use the Windows commands for reboot or shutdown: either from the top right corner of the full-screen start menu, or by right-clicking on the start button and using the context menu.

When rebooting an individual server, Windows can be used exclusively. If the server is shut down, vCenter must be used to restart it (see section 3.1.2.2.2 Power on/off, startup/shutdown), as that is the only way to access powered-off virtual machines.

If the entire BHS needs to be shut down, the order of powering off VMs is not important, but there are some notes for rebooting (see also the next section). The only important dependency between virtual machines is that vCenter needs the domain to be active before it can fully boot up. Consequently, manual reboots should power off the primary domain controller (BHS-DC01) last and boot it up first.

If vCenter is not available (because it has been shut down), the vSphere thick client can be used to connect directly to one of the host servers (using local credentials) as noted in the topic,section 3.1.2.1.1 Logging In. Power options are shown in the topicsection 3.1.2.1.2 View Information about a VM.

The virtual machines are shown in a tree similar to the Web Client and can be powered on in much the same fashion. Only the virtual machines associated with a host are seen in the vSphere client for that host (not the full system). Connect to each host in turn until the correct VM is located if the state was not observed before shutdown.

The order of shutting down virtual servers is outlined below (there is no need to wait in between each). Reboots can follow in the opposite order (bottom-up, also no need to wait between starting machines):

  1. BHS-DC02
  1. BHS-HMI02
  1. BHS-HMI01
  1. BHS-DHS
  1. BHS-SAC
  1. IPM VM
  1. vCenter
  1. BHS-DC01
