Server Hardware Monitoring

The PRTG monitoring application is installed on the DHS. This software uses Simple Network Management Protocol (SNMP) and other technologies to interrogate devices in the BHS to determine their status. These statuses are then displayed on the HMI as red or green indicator lights on the Watchdog portion of the Communication Status screen. This status information is by no means exhaustive (its only states are Failed and Working), but it serves as an early problem indicator for troubleshooting. The monitored devices include:

  • SmartSort – Monitors the General Health of all the SmartSort Services on the SAC and DHS. If any service is not running, the associated indicator will display red and an alarm will be displayed.
  • Resources – Monitors the Hard Drive Space, CPU Usage, and Free Memory on all the Servers. If the usage is less than the threshold, an alarm will be displayed.
  • Power – Monitors the two server rack UPSs. If the system loses power and uses the battery, the associated indicator will display red and an alarm will be displayed. The current load percentage and battery time remaining are also periodically updated for each UPS.
  • Network – Monitors the Network Switches and the server NICs. If a network device fails or is unplugged, the associated indicator will display red and an alarm will be displayed.
  • VMware – Monitors the Health of the VMware Hot Backup Solution. If server redundancy is compromised, the associated indicator will display red and an alarm will be displayed.
  • Hardware – If any of the physical servers experience a failure detected by the HP on-board software, the associated indicator will display red and an alarm will be displayed.
  • Connectivity – If PRTG detects that a device loses connectivity, it will stop monitoring the sensors associated with the device and display a PING fault. This will raise the connectivity alarm on the HMI.
  • SAN – The SAN storage device is monitored for hard drive and other internal failures. Faults with the SAN will be shown as an HMI alarm.

All the HMI Watchdog alarms have a note to see the Dashboard. Each fault that occurs is only one indicator of a problem. The System Dashboard displays details of the servers that are affected by the problem. From there, troubleshooting can continue to that server or the software that monitors the device.

The PRTG application itself also has a web application that can be reached from the Links screen. This page shows the current status of all sensors in the system.

Some servers may have duplicate monitoring for faults if they are shut down for an extended period of time (e.g. servers other than the SAC and DHS that don't have built-in alarms).
