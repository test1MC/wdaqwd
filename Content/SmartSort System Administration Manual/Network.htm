Network

The following sections outlineoutlines the common network issues, and the probable causes and remedies for those issues.

Lost Connection to Single Device on Network

Cable, connection, or power problem:

  1. The Ping command is the easiest troubleshooting tool. If a ping is successful, there are no low-level connectivity problems to the device.
  1. Check for lights on the device network card and any other network devices upstream from the device (fiber transceivers or switches). Lights should be on and blinking if there is activity.
  1. Trace the cables between two devices. This includes connections at junctions or switches. Reset all connectors and wait 30 seconds for lights to appear and turn green.
Lost Connection to Multiple Devices

Depending on network segmentation, this indicates a switch or hub problem:

  1. Find the common connection point to all disconnected devices on a network drawing. Troubleshoot that device first using the techniques above.
  1. Use a Maintenance Laptop to confirm connectivity along the path. Some MCPs and have Ethernet ports for this purpose.
General Network Communication Faults (One or More Devices)

Some devices are disconnected or are not powered:

  1. Find a computer with connectivity.
  1. Ping the PLC addresses from the workstation to make sure they are available

Ethernet cables loose:

  • Check Ethernet cables connections on PC – Cisco Switch – PLC Chassis.

Ethernet card not working on PC:

  1. Check if activity lights are on.
  1. Check the Windows Event Log to determine if network card has failed.

Ethernet module not working on PLC Chassis:

  1. Check diagnostic lights on Ethernet Module.
  1. Reset Ethernet Module or cycle power on the rack.
  1. Check if activity lights are on Ethernet card. Confirm status or check for errors on front display.

Confirm redundancy of switches. One switch may be down, but system will still be functional:

  • Check if the core switches are both running and that the status lights are active for the connected Ethernet ports.

Faulty Cisco Switch; power supply or fan may fail (rare):

  1. Basic functions will not work. Toggle power first to reboot the switch to see if that clears the errors.
  1. Replace Switch and reconfigure.
