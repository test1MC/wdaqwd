Application Overview

The general architecture of the system is described in the topics listed belowsections that follow. For more detail, including all network connections, please refer to the Network Drawings provided with this project. Screenshots are for illustrative purposes. Some screens may vary from the actual installation on site.

The basic architecture of the system is to have virtual machines for all server computers that run the BHS. These servers are hosted by two physical machines located in the server rack. All physical disk storage for these VMs is provided by a Storage Area Network (SAN), also in the server rack. Redundancy is provided by the VMware vSphere software.

The design of the upper-level system is centered on three components: database, services, and applications. The database is the core repository of information for the BHS and it resides on the SAC. For systems with a secondary server (DHS), the SAC database copies its information to the DHS, and only retains configuration data and recent history. Reports, queries, and non-real-time processes are run against the secondary server (DHS) which reserves the SAC for real-time transactional message processing and sortation.

The SmartSort services are installed on the servers and provide all the automatic baggage handling functionality described in this manual.

The Web Client application is accessible from the operator workstations to provide the means for operators to change system settings, manage flight schedules, and query information in the historical data tables.
