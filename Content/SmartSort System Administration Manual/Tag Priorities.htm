Tag Priorities

The priority of the different barcode types that can be read by the ATRs in the system is configured using the Tag Priorities screen. This information directly impacts the sort logic because it determines which barcode types will be used by the system, and how the system will sort a bag that has more than one barcode type.

Figure 2‑20: Tag Priorities Screen

Tag types cannot be added or deleted. Likewise, the description and pattern of a tag cannot be altered. When a tag type is selected from the Tag Priority List, its details are displayed in the Tag Type Details section on the right side of the screen, and the following configurations are available:

  • Tag Priority – The priority in which tag data is processed can be configured. If there is more than one tag on a bag, the tags that have been configured with the highest priority are used to determine the bag's sort destination. The lower the priority number, the higher the priority (i.e. setting the "Priority" field to 1 will cause that tag type to be of the highest priority).
  • Enable/Disable Tag Types – Tags that are enabled will be read by the ATR and the data from the barcode will be used to determine the bag's sort destination. If the Enabled checkbox is not checked for that tag type, however, those tags will be ignored by the sortation logic.

Making a change to the tag priority settings sends a message to the SAM service on the SAC and to the MEC application that a change has been made to the tag priority list. The list will automatically be reloaded from the database.
