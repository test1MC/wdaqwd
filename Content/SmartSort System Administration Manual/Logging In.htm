Logging In

A valid User Name and Password are required to use the vSphere Web Client application. Anyone that needs to have access to this application is assigned a unique user ID and password.

Each user is also assigned to one or more user groups. User groups are used to categorize users who perform similar functions (e.g. supervisors, operators, etc.). Each user group has access to particular screens and functions. These permissions can be adjusted at any time (typically by an administrative user). Any changes made to a user group affects all members of that group.

Once the login is successful, the user has access to any screens and functions for which they have been granted permission.

Logging In

The vSphere Web Client can be accessed from the following URL: <DFW-D-VCSA.smartsuite.dfwd>. The Web Client should be accessible from any web browser, but is works best with Google Chrome (installed on the Maintenance Server).

When the vSphere Web Client is first accessed, the sign-on screen is displayed.

Figure 3‑5: Sign-On Screen

To log into the vSphere Web Client:

  1. Enter the user name in the User name field.
  1. Enter the password in the Password field.
  1. Click Login.

To log out of the vSphere Web Client:

  1. Click the user name at the top of the screen. The user drop–down menu displays.
  1. Click Logout.
