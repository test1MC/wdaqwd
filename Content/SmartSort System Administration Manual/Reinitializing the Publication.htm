Reinitializing the Publication

To Reinitialize the Subscribers:

  1. Mark the Subscription for Reinitialization: Instructs SQL Server to stop replicating, and requires that a fresh snapshot be applied to the subscriber.
  1. View the Snapshot Agent: Generates the new snapshot.
  1. Start the Synchronization: Copies the snapshot to the subscriber, and restarts the replication process.

Mark Subscription for Reinitialization:

  1. Select the publication in the server tree.


  1. Right-click the subscription, and click Reinitialize from the menu
  1. Select the Use a new snapshot radio button, and click the Mark for Reinitialization button.

View the Snapshot Agent:

The Snapshot starts generating in the background. The status of the snapshot can be viewed using the "View Snapshot Agent Status" from the Publication context menu in SQL Management Studio. The Reinitialization process removes any data on the DHS that already exists on the SAC in the larger data tables (bag records, transactions, host messages). The snapshot then copies all the SAC data to the DHS.

Start the Synchronization

Once the snapshot is complete, watch the Replication Monitor to make sure that it is running smoothly. SQL Server attempts to restart the replication process whenever it fails. As it is retrying, the Replication Monitor status icons change from Failed/Error to OK. It is therefore important to watch the replication monitor for about five minutes to make sure that it is indeed running. The screen refreshes automatically, but can also be refreshed manually using the F5 key.
